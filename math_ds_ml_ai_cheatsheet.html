Basic Probability
P(A)Probability of event A, between 0 and 1
P(A or B)= P(A) + P(B) − P(A and B)
P(A and B)= P(A)·P(B) if independent
P(A|B)Probability of A given B already happened
oddsP(A) / (1−P(A)) — used in logistic regression's logit
Shows up in: classifiers, A/B tests, uncertainty
Python
# estimate P(A), P(A and B) from data
import numpy as np

clicked = np.array([1, 0, 1, 1, 0])
bought  = np.array([1, 0, 0, 1, 0])

p_clicked = clicked.mean()             # P(A)
p_both = (clicked & bought).mean()      # P(A and B)
p_bought_given_clicked = p_both / p_clicked  # P(B|A)

Slopes & Basic Calculus
slope = Δy/ΔxRate of change — how fast one thing changes vs another
f'(x)Derivative — the slope at a single point
d/dx(xⁿ) = nxⁿ⁻¹Power rule — most derivatives you'll take reduce to this
minimumWhere the slope = 0 and curves upward — what training tries to find
Shows up in: intuition for "why does the model learn"
Python
# numerical derivative -- the slope at a point
def f(x):
    return x**2

def slope(f, x, h=1e-6):
    return (f(x + h) - f(x - h)) / (2 * h)

slope(f, 3)   # approx 6.0, matches f'(x)=2x
Reading ML Notation
θ (theta)Model parameters/weights, the things training adjusts
ŷ (y-hat)Predicted value vs y, the true/actual value
L(θ)Loss function — how wrong the model currently is
η (eta)Learning rate — step size during training
∇ (nabla)Gradient symbol — "the slope, but for many variables at once"
Shows up in: every paper and every framework's docs
Python
# theta (weights), y_hat, and a simple loss
theta = 0.5
x, y = 2.0, 3.0

y_hat = theta * x         # prediction

eta = 0.01                 # learning rate

Calculus for Optimization
∇fGradient — vector of partial derivatives, points uphill
chain ruled/dx f(g(x)) = f'(g(x))·g'(x) — the whole basis of backprop
σ(x) = 1/(1+e⁻ˣ)Sigmoid — squashes to (0,1), σ' = σ(1−σ)
ReLU(x)max(0,x) — derivative is 1 or 0, cheap to compute
θ ← θ − η∇LGradient descent update rule
Shows up in: training every neural network
Python
# one step of gradient descent, by hand
def loss(theta):
    return (theta - 3) ** 2

def grad(theta, h=1e-6):
    return (loss(theta + h) - loss(theta - h)) / (2 * h)

theta, eta = 0.0, 0.1
for _ in range(50):
    theta -= eta * grad(theta)   # theta -> ~3.0

Loss Functions
MSE(1/n)Σ(y−ŷ)² — standard regression loss
MAE(1/n)Σ|y−ŷ| — robust to outliers
binary cross-entropy−[y log ŷ + (1−y)log(1−ŷ)] — binary classification
categorical cross-entropyMulti-class version, paired with softmax
Shows up in: literally the thing your model is trained to minimize
Python
import numpy as np

y = np.array([1, 0, 1])
y_hat = np.array([0.9, 0.2, 0.6])

mse = np.mean((y - y_hat) ** 2)
mae = np.mean(np.abs(y - y_hat))

bce = -np.mean(y * np.log(y_hat) + (1 - y) * np.log(1 - y_hat))

Deep Learning & Research-Level Math

— the machinery behind Transformers, generative models, and modern optimizers
Matrix Decompositions
A = UΣVᵀSVD — works for any matrix; basis for PCA, LoRA, recommenders
A = QΛQ⁻¹Eigendecomposition — only for diagonalizable matrices
A = LLᵀCholesky — fast solves for positive-definite systems
A = QRQR decomposition — numerically stable least squares
tr(A), det(A)Trace (sum of eigenvalues), determinant (volume scaling)
Shows up in: PCA derivation, LoRA fine-tuning, Gaussian processes
Python
import numpy as np

A = np.random.rand(5, 3)

U, S, Vt = np.linalg.svd(A)         # SVD
L = np.linalg.cholesky(A.T @ A)      # Cholesky (needs PSD input)
Q, R = np.linalg.qr(A)                # QR
Multivariate Calculus
J (Jacobian)Matrix of partials for vector-valued functions — per-layer local derivative
H (Hessian)Matrix of second derivatives — curvature of the loss surface
∇f = λ∇gLagrange multipliers — constrained optimization (SVM margin)
convexityf(λx+(1−λ)y) ≤ λf(x)+(1−λ)f(y) — guarantees a global minimum
Shows up in: understanding why/when training gets stuck
Python
import torch

x = torch.tensor([1.0, 2.0], requires_grad=True)
f = (x ** 2).sum()

f.backward()
x.grad          # the gradient (Jacobian for scalar output)

# Hessian via autograd
h = torch.autograd.functional.hessian(lambda x: (x**2).sum(), x.detach())
Information Theory
H(P) = −ΣP(x)logP(x)Entropy — average uncertainty in a distribution
H(P,Q)Cross-entropy — literally your classification loss
KL(P‖Q)= H(P,Q) − H(P); divergence between two distributions
perplexity2^H or e^(cross-entropy) — standard LM quality metric
Shows up in: VAEs, distillation, RLHF's KL penalty, LM evaluation
Python
from scipy.stats import entropy
import numpy as np

p = np.array([0.5, 0.5])
q = np.array([0.9, 0.1])

entropy(p)          # H(P)
entropy(p, q)        # KL(P || Q)

perplexity = np.exp(entropy(p))
MLE & Bayesian Inference
θ* = argmax Σ log P(xᵢ|θ)Maximum likelihood estimation
P(θ|D) ∝ P(D|θ)P(θ)Bayes' rule — posterior ∝ likelihood × prior
MSE ⟺ MLESquared error is MLE under a Gaussian noise assumption
cross-entropy ⟺ MLECross-entropy is MLE under a categorical assumption
Shows up in: justifying why "standard" losses are the ones we use
Python
from scipy.optimize import minimize
import numpy as np

data = np.array([2.1, 1.9, 2.3, 2.0])

# MLE for a Gaussian mean: minimize negative log-likelihood
def neg_log_lik(mu):
    return -np.sum(-0.5 * (data - mu) ** 2)

result = minimize(neg_log_lik, x0=0.0)
result.x   # ~ sample mean

Transformer & Generative Math

Python
import torch
import torch.nn.functional as F

Q, K, V = torch.randn(1,4,8), torch.randn(1,4,8), torch.randn(1,4,8)
d_k = Q.shape[-1]

scores = Q @ K.transpose(-2, -1) / d_k**0.5
attn = F.softmax(scores, dim=-1)
output = attn @ V   # scaled dot-product attention
