Core Syntax

select · where · order by
Selecting Data
SELECT col1, col2 FROM table;Select specific columns from a table
SELECT * FROM table;Select every column from a table
SELECT DISTINCT col FROM table;Select only unique values from a column
SELECT col FROM table LIMIT 10;Limit the number of rows returned (Postgres uses LIMIT instead of TOP)
Filtering & Ordering
WHERE conditionFilter rows before grouping or aggregation
WHERE a AND b / OR b / NOT aCombine or negate multiple conditions
ORDER BY col ASC | DESCSort the result set by one or more columns
col AS aliasGive a column or table a temporary name for the query
Comments
-- single line commentComment out the rest of a line
/* multi-line comment */Comment out a block spanning multiple lines

Joins & Unions

inner · left · right · full
Joins
INNER JOINReturn only rows with matches in both tables
LEFT JOINReturn all left-table rows, with matches from the right where available
RIGHT JOINReturn all right-table rows, with matches from the left where available
FULL JOINReturn all rows from both tables, matched or not
Self Join — Example
SELECT a.name AS employee, b.name AS manager FROM employees a JOIN employees b ON a.manager_id = b.id;
Combining Result Sets
UNIONCombine result sets and remove duplicate rows
UNION ALLCombine result sets and keep all rows, including duplicates

Grouping

group by · having · exists
Grouping
GROUP BY colGroup rows that share the same value for aggregation
HAVING conditionFilter grouped rows after aggregation, unlike WHERE
Subquery Conditions
EXISTS (subquery)True if the subquery returns at least one row
NOT EXISTS (subquery)True if the subquery returns no rows
col = ANY (subquery)True if the condition matches at least one subquery row
col = ALL (subquery)True only if the condition matches every subquery row

CTEs & Subqueries

with · recursive
Common Table Expressions
Basic CTE
WITH recent AS ( SELECT * FROM table WHERE date >= '2025-01-01' ) SELECT * FROM recent;
Multiple, chained CTEs
WITH a AS (SELECT ...), b AS (SELECT ... FROM a) SELECT * FROM b;
Recursive CTE
WITH RECURSIVE org AS ( SELECT id, manager_id FROM employees WHERE manager_id IS NULL UNION ALL SELECT e.id, e.manager_id FROM employees e JOIN org ON e.manager_id = org.id ) SELECT * FROM org;
Subqueries
FROM (SELECT ...) AS subUse a query's result as a virtual table
WHERE col IN (SELECT ...)Filter rows using values returned by a subquery
SELECT col, (SELECT ...) AS aliasUse a scalar subquery to return one value per row

Date Filtering

WHERE / HAVING clause
Exact Period Match
date_trunc('month', date) = '2022-04-01'Filter rows to a single calendar month
to_char(date, 'YYYY-mm') = '2023-05'Filter rows to a month using formatted text
to_char(date, 'YYYY') = '2023'Filter rows to a single year using formatted text
HAVING min(date_trunc('year', date)) = '2021-01-01'Filter groups whose earliest year matches a given value
Date Ranges
date >= '2023-5-1' AND date < '2023-6-1'Filter rows within an explicit date range
date >= '2025-01-01' AND date < '2026-01-01'Filter rows to a full calendar year using bounds
date_trunc('day', date) >= d AND < d + interval '1 day'Filter rows to one specific calendar day
date > sd.first_activityFilter rows to after a reference date from another table
Rolling Windows
date(date) BETWEEN (getdate() - interval '1 year') AND getdate()Filter to a rolling trailing 12 months
date >= dateadd(year, -2, getdate())Filter to a rolling trailing 2 years
Full Example
WHERE date_trunc('day', date) >= ('2023-01-17'::date + interval '0 hour') AND date_trunc('day', date) < ('2023-01-17'::date + interval '0 hour' + interval '1 day')

Window Functions

over · partition by
Window Function Basics
func() OVER (PARTITION BY col ORDER BY col2)Apply a function across a defined "window" of related rows
ROW_NUMBER() OVER (...)Assign a unique sequential number to each row within its partition
NTILE(4) OVER (ORDER BY col)Split rows into a fixed number of roughly equal buckets
FIRST_VALUE(col) OVER (...)Return the first value in the current window frame
LAST_VALUE(col) OVER (...)Return the last value in the current window frame
Frame Clauses
ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROWFrame from the start of the partition to the current row
ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWINGFrame covering the current row plus one row on each side
RANGE BETWEEN ...Frame based on value ranges rather than physical row position

Running Totals

window functions
Cumulative Sums
sum(revenue) OVER (...) AS running_revenueRunning total of revenue up to the current row
sum(bet) OVER (...) AS sum_betRunning total of stakes up to the current row
sum(quantity) OVER (...) AS running_quantityRunning total of quantity up to the current row
sum(amount_a_eur) OVER (...)Running total of amount_a converted to EUR, up to the current row
Full Example
SELECT date, SUM(revenue) OVER ( ORDER BY date ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW ) AS running_revenue, SUM(quantity) OVER ( ORDER BY date ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW ) AS running_quantity FROM table;

Lag, Lead & Rank

window functions
Row Comparisons
LAG(revenue) OVER (ORDER BY date)Return the previous row's revenue value
LEAD(revenue_in_eur, 12) OVER (ORDER BY month)Return revenue from 12 periods ahead
RANK() OVER (PARTITION BY category ORDER BY revenue DESC)Rank rows within each category by revenue
LAG(date) OVER (ORDER BY date) AS prev_activityReturn the previous row's date
Time-Gap & Span Calculations
EXTRACT(epoch FROM (date - LAG(date) OVER (...))) / 60.0Minutes elapsed since the previous row
MIN(date), MAX(date)Earliest and latest dates within a group
EXTRACT(epoch FROM (MAX(date) - MIN(date))) / 3600.0Total hours spanned by a group of rows

Growth & YoY

period-over-period
Absolute & Percentage Growth
revenue - LAG(revenue) OVER (ORDER BY month)Absolute change in revenue vs. the previous period
(rev - LAG(rev)) / LAG(rev) * 100Percentage change in revenue vs. the previous period
LAG(revenue) OVER (ORDER BY month) AS previous_month_salesPrevious month's revenue, for side-by-side comparison
Full Example — Formatted Growth %
SELECT month, sum(revenue) AS revenue, ROUND( 100 * (sum(revenue) - lag(sum(revenue), 1) OVER (ORDER BY month)) / lag(sum(revenue), 1) OVER (ORDER BY month) , 0) || '%' AS growth FROM sales GROUP BY month;
